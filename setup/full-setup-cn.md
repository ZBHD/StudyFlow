## 第 1 步：复制文件（5 分钟）

### Git 用户（推荐）

```bash
# 把 StudyFlow 作为 submodule 引用
cd your-study-folder
git submodule add https://github.com/ZBHD/StudyFlow.git .studyflow

# 复制核心文件到你的目录
cp -r .studyflow/core .
cp -r .studyflow/workflow .
```

> **为什么用 submodule？** StudyFlow 更新时，你只需要 `git submodule update` 就能获得最新版本。

### 非 Git 用户

直接把 `core/` 和 `workflow/` 目录复制到你的学习文件夹。

## 第 2 步：建立你的画像（10 分钟）

### 2.1 让 AI 收集信息

打开你的 AI 工具，说：

```
我想建立学习画像。请你通过对话收集我的信息。

需要收集：
1. 基本情况（职业/年级、专业、学习目标）
2. 时间预算（每周可用时间、偏好时间段）
3. 学习习惯（喜欢什么学习方式、容易分心吗）
4. 当前水平（学过什么、掌握程度）
5. 具体目标（考试、项目、求职）

请你一问一答，帮我整理成结构化的画像。
```

### 2.2 填写画像

AI 会问你一系列问题，如实回答。AI 会帮你生成结构化的画像。

**示例输出：**

```markdown
## 基本情况
- 身份：大学生
- 专业：计算机科学
- 目标：考研 + 技术能力提升

## 时间预算
- 每周可用：20 小时
- 偏好：上午 8-12 点

## 当前水平
- 编程：Python 基础、C 语言入门
- 数学：高数在学
- 专业课：数据结构开始
```

### 2.3 写入画像文件

把 AI 生成的内容写入 `core/user/profile.md`。

## 第 3 步：建立能力评估（10 分钟）

### 3.1 让 AI 评估你的能力

说：

```
现在帮我做能力评估。

评估维度：
1. 编程能力（语言、算法、项目）
2. 数学能力（高数、线代、概率论）
3. 专业知识（数据结构、操作系统、计算机网络）
4. 学习能力（理解速度、自学能力、坚持度）

每个维度给出 0-5 分，并说明理由。
```

### 3.2 写入能力评估

把 AI 生成的评估写入 `core/user/ability-map.md`。

## 第 4 步：配置 AI 行为（5 分钟）

### 4.1 设置角色

告诉 AI：

```
从现在开始，你有两个角色：

Leader（规划者）：
- 负责制定计划、监督执行、复盘总结
- 要求严格，但不人身攻击
- 批评行为，不批评人格
- 说人话，别废话

Teacher（老师）：
- 负责讲解知识、答疑解惑
- 用类比和引导，不直接给答案
- 给提示，让我自己想
- 我试过之后再给答案

你先读一下 core/user/ai-rules.md，这是你的行为规范。
```

### 4.2 验证配置

AI 应该回答：

- 它理解了两个角色的区别
- 它会按规范执行
- 它知道怎么处理违规

如果 AI 没理解，重复说明或修改 `core/user/ai-rules.md`。

## 第 5 步：制定学习计划（15 分钟）

### 5.1 确定阶段

告诉 AI：

```
帮我制定学习计划。

我目前的情况：
- 大一，计算机专业
- 目标：考研（3 年后）
- 当前进度：数据结构刚开始

请帮我：
1. 把 3 年分成几个阶段
2. 每个阶段的学习重点
3. 阶段之间的衔接
```

### 5.2 制定周计划

```
这是第一阶段，帮我制定第一周的计划。

要求：
- 每天 2-3 小时
- 上午 8-12 点学习
- 重点：数据结构
- 包含复习时间
```

### 5.3 写入计划文件

把 AI 生成的计划写入：

- `workflow/planning/current-stage.md` — 当前阶段
- `workflow/planning/weekly-plan.md` — 周计划

---

## 第 6 步：设置日常流程（5 分钟）

### 6.1 每日流程

每天学习后，告诉 AI：

```
今天学了：
- 内容：（学了什么）
- 时间：（学了多久）
- 收获：（学到了什么）
- 问题：（遇到什么困难）
```

AI 会：
- 记录到日记
- 更新进度
- 提醒复习
- 调整计划

### 6.2 每周复盘

每周日，告诉 AI：

```
这周复盘：
- 计划完成度：（完成了多少）
- 最大收获：（学到了什么）
- 最大问题：（遇到什么困难）
- 下周计划：（想怎么安排）
```

AI 会：
- 分析问题原因
- 调整下周计划
- 更新能力评估
- 安排复习

---

## 第 7 步：设置复习系统（5 分钟）

### 7.1 让 AI 管理复习

告诉 AI：

```
帮我设置复习系统。

规则：
- 学完新知识后，D+1 复习第一次
- D+3 复习第二次
- D+7 复习第三次
- D+14 复习第四次
- 间隔逐渐拉长

每次复习时，你问我几个问题，检验我记住了多少。
```

### 7.2 记录错题

做错的题目，告诉 AI：

```
这道题我错了：
- 题目：（题目内容）
- 我的答案：（我的回答）
- 正确答案：（正确答案）
- 错误原因：（为什么错）
```

AI 会：
- 记录到错题本
- 分析错误模式
- 安排错题复习

## 常见问题

**Q: 我应该每天和 AI 聊多久？**
A: 10-30 分钟。主要是汇报进度、问问题、让 AI 调整计划。

**Q: AI 给的计划不好怎么办？**
A: 告诉它哪里不好，让它调整。AI 是你的搭档，不是你的老板。

**Q: 我忘记汇报了怎么办？**
A: 补上就行。告诉 AI 你昨天学了什么，它会帮你记录。

**Q: 我想换一个 AI 工具怎么办？**
A: 直接换。把你的画像发给新 AI，它就能继续帮你。

**Q: 我想分享我的学习记录怎么办？**
A: 直接把 `workflow/diary/` 目录分享出去。这是纯 Markdown，任何人都能看。
